Redesign Settings: IA + Accessibility Focus

Redesigning settings to reduce cognitive load and improve accessibility

Information Architecture Accessibility Vue/SASS User Research

Problem

  • User settings page was a single row with no organization
  • Users struggled to find settings; conflicting internal opinions existed on where info should live
  • Cognitive overload due to too much information on one page
  • Users had to reload page to change settings for a different organization
  • Developer-driven design with little human-centered thought

Research

  • Analyzed Reddit discussions on best practices for settings organization and structure
  • Reviewed internal support tickets and gathered direct user feedback to identify pain points
  • Conducted surveys and interviewed Solutions Engineers who work directly with users

Approach

Key Innovation

Implemented a progressive disclosure pattern ("black box" concept) specifically designed for neurodivergent users. Category names remain visible while settings reveal on-demand, reducing cognitive overload while maintaining context and discoverability.

Process

  1. Analyzed current API response and compared to competitor implementations
  2. Mapped ideas on a whiteboard and grouped settings into logical categories
  3. Iterated on concise category names for clarity and scannability
  4. Created wireframes progressing from whiteboard sketches to notebook iterations to Sketch high-fidelity designs
  5. Built interactive prototypes to validate information architecture and interaction patterns
  6. Conducted accessibility testing including color contrast checks, screenreader testing with VoiceOver, and validation with AxeDev Tools
  7. Made strategic technical decisions including component architecture in Vue, state management for settings persistence without page reloads, and SASS organizational patterns for maintainable, scalable styles
  8. Implemented frontend in Vue/SASS with full accessibility attributes
  9. Tested in production environment via feature flags with internal users before wider rollout

Results

Impact
  • Redesigned information architecture reduced user confusion and support burden; users could now locate settings in a single organized view instead of scanning unstructured lists
  • Achieved comprehensive accessibility compliance through iterative testing with screenreaders and AxeDev Tools, ensuring settings were fully navigable for users with visual, motor, and cognitive disabilities
  • Significantly reduced cognitive load by implementing progressive disclosure, allowing users to focus on relevant settings without overwhelming information
  • Received positive feedback from users and internal teams; fewer admin cleanup tasks needed
  • Successfully addressed legacy technical debt and usability issues
Tools & Methods
  • Whiteboard ideation and pen & paper sketching
  • Sketch for high-fidelity wireframes and prototypes
  • AxeDev Tools and VoiceOver screenreader testing
  • Vue/SASS for component implementation
  • Feature flags for safe production testing

Messaging Modernization

Transforming a cluttered chat system into a streamlined, accessible experience

UX Design Accessibility Vue/SASS Storybook

Problem

  • Users were not using threads effectively, resulting in lost context within the application
  • Messaging system lacked modern chat features expected by users familiar with contemporary platforms
  • Confusing layouts, missing features, and inconsistent UX patterns created friction
  • Existing design did not align with the upcoming application-wide redesign

Research

  • Conducted in-depth interviews with Solutions Engineers who support users daily
  • Analyzed customer feedback and patterns from support ticket submissions
  • Performed comparative analysis of competitor chat systems including Slack, Discord, and Microsoft Teams
  • Facilitated user discussions and distributed surveys to understand feature expectations

Approach

Key Innovation

Conducted comparative analysis of modern chat systems (Slack, Discord, Teams) to identify user expectations and established patterns, then strategically adapted these patterns to fit existing technical constraints while maintaining comprehensive accessibility standards throughout.

Process

  1. Created wireframes progressing from initial whiteboard sketches to refined Sketch high-fidelity designs
  2. Designed and integrated modern features including message quoting, threaded replies, context blocks, and simplified navigation menus
  3. Conducted thorough accessibility validation including color accessibility checks, comprehensive screenreader testing, and AxeDev Tools auditing
  4. Built and documented reusable message components in Storybook, establishing patterns for quotes, replies, and thread contexts that could scale across the application while maintaining accessibility attributes
  5. Implemented frontend components in Vue/SASS with full ARIA support and semantic HTML
  6. Deployed via feature flags for controlled internal testing before production release
  7. Made iterative refinements based on testing feedback, addressing edge cases for sizing, long text handling, system messages, and screenreader announcement patterns

Results

Impact
  • Messaging interface became significantly more organized with clear visual hierarchy and intuitive interaction patterns
  • Users gained efficient ways to quote messages, reply in context, and access conversation history, reducing confusion and improving productivity
  • Reduced administrative overhead by minimizing cleanup tasks and support requests related to messaging
  • Successfully aligned messaging experience with the upcoming application redesign, creating visual and interaction consistency
  • Improved accessibility compliance and better matched user expectations established by modern communication platforms
Tools & Methods
  • Whiteboard ideation and collaborative sketching
  • Sketch for high-fidelity design and prototyping
  • AxeDev Tools and VoiceOver screenreader testing
  • Vue/SASS for component implementation
  • Storybook for component documentation and design system
